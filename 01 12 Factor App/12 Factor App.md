The Twelve Factors  
[2]

[I. Codebase](https://12factor.net/codebase)  
One codebase tracked in revision control, many deploys[2]

[II. Dependencies](https://12factor.net/dependencies)  
Explicitly declare and isolate dependencies[2]

[III. Config](https://12factor.net/config)  
Store config in the environment[2]

[IV. Backing services](https://12factor.net/backing-services)  
Treat backing services as attached resources[2]

[V. Build, release, run](https://12factor.net/build-release-run)  
Strictly separate build and run stages[2]

[VI. Processes](https://12factor.net/processes)  
Execute the app as one or more stateless processes[2]

[VII. Port binding](https://12factor.net/port-binding)  
Export services via port binding[2]

[VIII. Concurrency](https://12factor.net/concurrency)  
Scale out via the process model[2]

[IX. Disposability](https://12factor.net/disposability)  
Maximize robustness with fast startup and graceful shutdown[2]

[X. Dev/prod parity](https://12factor.net/dev-prod-parity)  
Keep development, staging, and production as similar as possible[2]

[XI. Logs](https://12factor.net/logs)  
Treat logs as event streams[2]

[XII. Admin processes](https://12factor.net/admin-processes)  
Run admin/management tasks as one-off processes[2]
